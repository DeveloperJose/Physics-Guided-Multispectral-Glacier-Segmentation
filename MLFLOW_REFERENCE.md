# MLflow Integration Reference

## Overview

MLflow is used at three layers:

| Layer | File(s) | Purpose |
|---|---|---|
| Training-time | `scripts/train.py`, callbacks | Real-time metric/artifact logging via `MLFlowLogger` |
| Post-training upload | `scripts/upload_to_mlflow.py` | Batch upload runs from disk; optionally regenerate visualizations |
| Plotting | `visualization/mlflow_run_plots.py` | Fetch metric histories, produce training curves |

**Default tracking URI:** `https://mlflow.josegperez.com/`  
**Disable:** `--mlflow-enabled false` on `train.py`

## Key Workflows

### Initial Upload
```bash
uv run python scripts/upload_to_mlflow.py output/my_run_name --server desktop
uv run python scripts/upload_to_mlflow.py --batch --server desktop
uv run python scripts/upload_to_mlflow.py --batch --experiment-type baseline_ci --server desktop
```

### Regenerate Visualizations
```bash
uv run python scripts/upload_to_mlflow.py output/my_run_name --server desktop --regenerate
uv run python scripts/upload_to_mlflow.py --batch --server desktop --regenerate
uv run python scripts/upload_to_mlflow.py output/my_run_name --server desktop --regenerate --high-res --val-viz-n 8 --test-eval-n 8
```

### Training Without MLflow
```bash
uv run python scripts/train.py --config configs/desktop/debris_ice/bs8.yaml --server desktop --gpu 0 --mlflow-enabled false
```

### Plotting from MLflow
```bash
uv run python visualization/mlflow_run_plots.py --run-name "reproducibility_dci_02_landsat_dem_seed42_desktop_20260608_152403"
```

## Metric Naming Convention

All metrics follow: `{scope}_{split}_{target}_{metric}`

| Scope | Meaning |
|---|---|
| `window_*` | Per-batch metrics during training (lightning loops) |
| `full_*` | Full-split prediction pipeline with postprocessing |
| `best_full_*` | Best checkpoint's full-split metrics |

Examples: `window_train_ci_iou`, `window_val_dci_iou`, `full_test_ci_iou`, `best_full_test_dci_iou`.

## Experiment & Run Naming

- **Experiment name:** Set via `training_opts.mlflow_experiment_name` in config, falls back to `{prefix}_{task}` (e.g. `reproducibility_debris_ice`).
- **Run name:** `{base_run_name}_{server}_{YYYYMMDD_HHMMSS}` (auto-generated by `mlflow_utils.generate_run_name()`).
- **Run deduplication:** Upload script searches by `tags.mlflow.runName`. 0 matches → create, 1 match → skip (must use `--regenerate`), >1 → error (clean up duplicates in MLflow UI).

## What Gets Logged

| Data | When | Artifact Path |
|---|---|---|
| Params & scalar metrics | Training loop | (top-level) |
| Checkpoints (`*.ckpt`) | Initial upload | `checkpoints/` |
| Config (`conf.json`) | Initial upload | `config/conf.json` |
| Training log | `on_fit_end` | `logs/training.log` |
| Visualizations | During training + `--regenerate` | `val_visualizations/`, `test_evaluations/` |
| CSV metrics | During training | `csv_metrics/` |
| Errors/warnings | On exception | `errors/`, `warnings/` |

## Key Module: `glacier_mapping/utils/mlflow_utils.py`

| Function | Purpose |
|---|---|
| `categorize_experiment(config)` | Determine MLflow experiment name from config |
| `extract_mlflow_params(config, server_config)` | Flatten config into param dict |
| `generate_run_tags(config, server_config, config_file)` | Generate searchable MLflow tags |
| `generate_run_name(base_run_name, server_name)` | Create unique run name with timestamp |
| `setup_mlflow_logger(tracking_uri, experiment_name, run_name, tags)` | Start MLflow run |
| `attempt_connection_with_retry(...)` | Retry MLflow connection on failure |
| `log_params_safe(params)` / `log_metrics_safe(metrics, step)` / `log_artifact_safe(local_path, artifact_path)` | Safe logging wrappers |

## Key Module: `scripts/upload_to_mlflow.py`

| Function | Purpose |
|---|---|
| `upload_single_run(run_dir, tracking_uri, server_name, ...)` | Upload one run |
| `upload_batch_runs(output_dir, tracking_uri, server_name, ...)` | Upload all runs in directory |
| `regenerate_visualizations(...)` | Reload checkpoint, regenerate PNGs + test metrics |
| `parse_tensorboard_logs(log_dir)` | Extract metrics from TB events file |
| `find_all_checkpoints(run_dir)` | List checkpoints sorted by epoch |

## Run Selection for Paper

Use `universal_prediction.py` for paired CI/DCI evaluation:
```bash
uv run python scripts/universal_prediction.py
```

This auto-pairs CI and DCI runs by base name and produces combined 3-class metrics.
